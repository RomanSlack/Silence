# Session Log — 2026-04-19

## Goal
Connect Chinese OpenBCI V3 clone board to Ubuntu desktop and verify EMG signal streaming (jaw clench test).

## Hardware
- **Board**: OpenBCI V3 compatible clone, labeled "ChipKIT v3-32" (32-bit PIC32), purchased on eBay
- **Cable**: Red CP210x USB-to-serial adapter with 3 wires (black/green/white), 3-pin connector on board end
- **Switch positions**: PC / OFF / BLE
- **Board LEDs**: Blue (DT/RB) and Red, both lit when powered on
- **Note**: eBay listing image shows 8-bit ATmega328P board — may differ from what was shipped

## What We Tried

### 1. Driver & Permissions
- Added user to `dialout` group via `sudo usermod -a -G dialout $USER`
- Confirmed `/dev/ttyUSB0` appears when cable plugged in (CP210x = USB ID `10c4:ea60`)
- Permissions fixed with `sudo chmod 666 /dev/ttyUSB0` (resets on every replug)

### 2. OpenBCI GUI
- Downloaded and ran official Linux GUI
- `/dev/ttyUSB0` never appeared in the serial port dropdown
- GUI error: "not being run with Administrator access"
- Root cause: likely a Java serial enumeration bug on Linux

### 3. BrainFlow Python (PC mode, wired)
- Installed BrainFlow 5.21.0
- `BoardIds.CYTON_BOARD` on `/dev/ttyUSB0` → error: "board doesn't send welcome characters"
- Raw serial probe at 115200/9600/57600 baud → 0 bytes received at all baud rates
- DTR assertion via `ser.setDTR(True)` caused the blue DT LED on board to light up — confirms electrical connection
- Tried DuPont wires on D11 (TX) and D12 (RX) per ChipKIT UART pinout — still 0 bytes

### 4. BLE Mode
- Switched board to BLE, scanned with `bleak` — board never appeared in scan
- Accidentally connected to a Levoit humidifier (LUH-M101S-WUS, `48:31:B7:81:50:56`) which was already paired
- Multiple scans with board ON vs OFF showed no consistent new device appearing
- BLE non-functional regardless of desktop/laptop power considerations

## Root Cause (Best Guess)
The board shipped with **custom firmware** that does not respond to standard OpenBCI protocol commands. The eBay listing explicitly states: *"this product cannot use the firmware linked above. Please contact the customer service and note your email. We will send the firmware to your email."*

Both the wired (zero bytes) and BLE (not advertising) failures are consistent with missing or incorrect firmware.
