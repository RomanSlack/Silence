# Session Log — 2026-04-19 (ML track, post-hardware-block)

## Goal
Start the ML backend while hardware is blocked on seller firmware. Build the full pipeline on the Gaddy & Klein 2020 public sEMG dataset so that once Roman's own recordings land, it's a data-source swap.

## What landed

- `ml_backend/` scaffolded — raw PyTorch (no Lightning), package layout at `silence_ml/`.
- Abstractions: `EMGRecording` dataclass, `EMGDataset` (Torch), swappable sources (`gaddy.py`, `openbci.py` stub).
- Preprocessing: Butterworth bandpass 20–500 Hz, 60 Hz notch, z-score, windowing with `window_sec`/`hop_sec`.
- Baseline model: 3-block 1D CNN (Conv→BN→GELU, strided), global average pool, linear head.
- Training loop: AdamW, CE loss, val-acc checkpointing to `checkpoints/best.pt`.
- Scripts: `download_gaddy.py`, `inspect_signal.py`, `dataset_stats.py`, `smoke_test.py`, `train_baseline.py`.
- Vocab files: `vocab/mvp_20words.txt` (Roman's target commands), `vocab/gaddy_closed_top20.txt` (months/days/am-pm — Gaddy's actual closed vocab).

## Runs

**Synthetic smoke test** — 6 classes, 360 windows — 100% val acc in 1 epoch. Pipeline shape correct.

**Gaddy closed_vocab + gaddy_closed_top20** — 4787 windows, 20 classes, 25 epochs — **33.7% val acc** (chance = 5%). Bottleneck is label noise (word-spotting on multi-word utterances like "Sunday May 21").

## Blockers & next steps

- **Gaddy is a poor vocabulary match** for Roman's MVP (20 commands: yes/no/up/down/…). We keep using it only to validate the architecture and filters. Real training waits on Roman's own isolated-word recordings.
- Next ML-side improvements when we pick it up:
  - Single-hit filtering or button-aligned word windows on Gaddy to push baseline higher and confirm architecture capacity.
  - Data augmentation: channel dropout, time-jitter, gain jitter.
  - Replace avg-pool head with temporal attention pooling to help localize the word within the window.
  - Start designing Model 2 (subvocal) self-distillation using Gaddy's silent_parallel_data + voiced_parallel_data pairs — Gaddy's parallel set is exactly the paired vocal/silent data we'd need to prototype the cascade before Roman records any subvocal data.
